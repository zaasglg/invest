# AI Чат-Бот для Turkistan Invest

## Описание

AI Чат-бот интегрирован в систему Turkistan Invest для помощи пользователям в получении информации о:
- Регионах и их инвестиционных показателях
- Инвестиционных проектах
- СЭЗ (Специальных экономических зонах)
- Индустриальных зонах
- Недропользователях
- Проблемных вопросах (Issues)
- Задачах и их статусах

## Установка и настройка

### 1. Получить API ключ Gemini

1. Перейдите на сайт [Google AI Studio](https://makersuite.google.com/app/apikey)
2. Войдите с помощью Google аккаунта
3. Нажмите "Create API Key"
4. Скопируйте полученный ключ

### 2. Добавить ключ в .env файл

Откройте файл `.env` и добавьте:

```env
GEMINI_API_KEY=ваш_api_ключ_здесь
```

### 3. Очистить кэш

```bash
php artisan config:clear
php artisan cache:clear
```

## Использование

### Виджет чат-бота

Чат-бот отображается в виде плавающей кнопки в правом нижнем углу каждой страницы (после авторизации).

1. **Открыть чат**: Нажмите на иконку робота в правом нижнем углу
2. **Задать вопрос**: Введите вопрос в поле ввода и нажмите Enter или кнопку отправки
3. **Очистить историю**: Нажмите на иконку корзины в заголовке чата
4. **Закрыть чат**: Нажмите на иконку X в заголовке

### Примеры вопросов

**О регионах:**
- "Сколько всего регионов?"
- "Какие проекты в Туркестанской области?"
- "Покажи статистику по регионам"

**О проектах:**
- "Какие проекты находятся в статусе реализации?"
- "Сколько архивных проектов?"
- "Покажи проекты с проблемами"

**О проблемах:**
- "Какие проблемные вопросы у региона [название]?"
- "Покажи все нерешенные проблемы"
- "Какие проблемы с высоким приоритетом?"

**О СЭЗ и индустриальных зонах:**
- "Список всех СЭЗ"
- "Какие индустриальные зоны есть?"
- "Проблемы в СЭЗ"

**О недропользователях:**
- "Покажи незаконных недропользователей"
- "Список недропользователей по региону"

## Архитектура

### Backend

**Контроллер:** `app/Http/Controllers/ChatController.php`
- `send()` - отправка сообщения и получение ответа
- `history()` - получение истории чата
- `clear()` - очистка истории

**Сервисы:**
- `app/Services/GeminiService.php` - взаимодействие с Gemini API
- `app/Services/ChatContextService.php` - построение контекста из базы данных

**Модель:** `app/Models/ChatMessage.php`
- Хранит историю чата пользователя

**Миграция:** `database/migrations/2026_04_08_051423_create_chat_messages_table.php`

### Frontend

**Компонент:** `resources/js/components/chat-widget.tsx`
- React компонент виджета чата
- Управление состоянием сообщений
- Отправка запросов к API

**UI компоненты:**
- `resources/js/components/ui/scroll-area.tsx` - область прокрутки

### API Endpoints

```
POST   /chat/send      - Отправка сообщения
GET    /chat/history   - Получение истории
DELETE /chat/clear     - Очистка истории
```

## Как работает бот

1. **Пользователь задает вопрос**
2. **Анализ запроса** - система определяет о каких сущностях идет речь (регионы, проекты, проблемы и т.д.)
3. **Сбор контекста** - из базы данных извлекаются релевантные данные
4. **Отправка в Gemini** - вопрос + контекст отправляются в Gemini API
5. **Получение ответа** - Gemini анализирует данные и формирует ответ на русском языке
6. **Сохранение** - вопрос и ответ сохраняются в историю чата

## Настройка и кастомизация

### Изменить prompt системы

Отредактируйте метод `buildSystemPrompt()` в `app/Services/GeminiService.php`

### Добавить новые типы данных

1. Добавьте определение в метод `analyzeQuery()` в `GeminiService.php`
2. Добавьте метод получения данных в `ChatContextService.php`
3. Добавьте обработку в метод `buildContext()`

### Изменить внешний вид

Отредактируйте компонент `resources/js/components/chat-widget.tsx`

## Безопасность

- API ключ хранится в `.env` и не коммитится в репозиторий
- Все запросы требуют авторизации (`auth` middleware)
- CSRF защита для всех POST/DELETE запросов
- Лимит длины сообщения - 1000 символов

## Ограничения API

- Gemini API имеет бесплатный лимит: 60 запросов в минуту
- Для production рекомендуется использовать платный план
- Максимальная длина ответа: 2048 токенов

## Troubleshooting

### Ошибка "API key not found"
- Проверьте, что `GEMINI_API_KEY` добавлен в `.env`
- Выполните `php artisan config:clear`

### Чат не отображается
- Убедитесь, что вы авторизованы
- Проверьте консоль браузера на ошибки JavaScript
- Выполните `npm run build`

### Бот не отвечает
- Проверьте логи: `storage/logs/laravel.log`
- Проверьте статус Gemini API
- Убедитесь, что API ключ валидный

## Поддержка

При возникновении проблем:
1. Проверьте логи Laravel: `storage/logs/laravel.log`
2. Проверьте консоль браузера
3. Убедитесь, что миграции выполнены: `php artisan migrate`
4. Пересоберите frontend: `npm run build`
